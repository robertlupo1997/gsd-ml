**Phase 1: Profile Dataset**

Steps:
1. Parse arguments: extract dataset_path (first arg) and target_column (second arg) from the skill invocation. Domain defaults to "tabular".
2. Verify dataset exists: `test -f {dataset_path}` -- abort with clear error if missing.
3. Profile the dataset via python -c:
```bash
python -c "
import json, pandas as pd
from gsd_ml.profiler import profile_dataset
from dataclasses import asdict
df = pd.read_csv('{dataset_path}')
profile = profile_dataset(df, '{target_column}')
print(json.dumps(asdict(profile), default=str))
"
```
4. Parse the JSON output. Extract: task, metric, direction, n_rows, n_features, missing_pct, leakage_warnings.
5. Display profile summary to user (print key stats).
6. If leakage_warnings is non-empty, display warnings prominently. Ask user if they want to continue (PROF-03).

**Phase 2: Scaffold .ml/ Directory**

Steps:
1. Generate run_id: `run-YYYYMMDD-HHMMSS` format.
2. Create .ml/ directory: `mkdir -p .ml/artifacts`
3. Write `.ml/config.json` with schema from research doc:
```json
{
  "run_id": "{run_id}",
  "domain": "tabular",
  "dataset_path": "../{relative_path_to_csv}",
  "target_column": "{target}",
  "task": "{from profile}",
  "metric": "{from profile}",
  "direction": "{from profile}",
  "budget_experiments": 25,
  "budget_minutes": 60,
  "start_time": "{ISO timestamp}"
}
```
4. Create `.ml/prepare.py` by reading the gsd_ml.prepare.tabular module source and writing it. Use python -c to get the source:
```bash
python -c "
import inspect
from gsd_ml.prepare import tabular
print(inspect.getsource(tabular))
"
```
Write this output to `.ml/prepare.py` (SCAF-02). This is the FROZEN data pipeline -- workflow must never edit it.
5. Create `.ml/train.py` by reading the template from `~/.claude/gsd-ml/templates/train-tabular.py`, replacing the 4 placeholder constants (__CSV_PATH__, __TARGET_COLUMN__, __METRIC__, __TASK__) with actual values from the profile, and writing to `.ml/train.py` (SCAF-03).
6. Create empty `.ml/results.jsonl` (touch).
7. Create empty `.ml/experiments.jsonl` (touch).
8. Compute baselines via python -c (TAB-04):
```bash
cd .ml && python -c "
import json, pandas as pd
from prepare import load_data, split_data, build_preprocessor
from gsd_ml.baselines.tabular import compute_baselines
df = load_data('{dataset_path}')
X_train, X_test, y_train, y_test = split_data(df, '{target_column}')
preprocessor = build_preprocessor(X_train)
X_proc = preprocessor.fit_transform(X_train)
baselines = compute_baselines(X_proc, y_train, '{task}', '{metric}')
print(json.dumps(baselines, default=str))
"
```
Display baseline results. These are the minimum bar the model must beat.
9. Create git branch: `git checkout -b ml/run-{run_id}` (SCAF-04).
10. Initial commit: `git add .ml/ && git commit -m "scaffold: initialize .ml/ for {run_id}"`

**Phase 3: Experiment Loop**

Instruct Claude Code to iterate:

Pre-loop setup:
- Initialize state: experiment_count=0, keep_count=0, revert_count=0, best_metric=None
- Record start_time from config.json

Each iteration:
1. **Guardrail check** (before each iteration):
   - Read config.json for budgets
   - Check: experiment_count >= budget_experiments -> STOP
   - Check: elapsed time (now - start_time) >= budget_minutes * 60 -> STOP
   - Check disk space: `python -c "import shutil; u=shutil.disk_usage('.'); print(u.free/(1024**3))"` -> if < 1.0 GB -> STOP
   - If any guardrail trips, print which one and break the loop (GUARD-01 through GUARD-04)

2. **Edit train.py** (LOOP-01):
   - Claude Code reads the current .ml/train.py
   - Claude Code edits it to try a different approach (new model, hyperparameters, features)
   - Provide guidance: "Try different model families (RandomForest, XGBoost, LightGBM, Ridge/LogisticRegression), hyperparameters, feature engineering. Each iteration should be meaningfully different."

3. **Run train.py** (LOOP-02):
   ```bash
   cd .ml && python train.py 2>.ml/train.log
   ```
   - Capture stdout. Parse the LAST line that contains valid JSON.
   - Extract metric_value and metric_name from the JSON.
   - If no valid JSON found, treat as error.

4. **Keep/Revert decision**:
   - Use DeviationHandler to decide. Call via python -c:
   ```bash
   python -c "
   import json
   from gsd_ml.guardrails import DeviationHandler
   handler = DeviationHandler({})
   decision = handler.handle(
       metric_value={metric_value},
       best_metric={best_metric_or_None},
       direction='{direction}',
       stderr=open('.ml/train.log').read()
   )
   print(json.dumps({'decision': decision}))
   "
   ```
   - **keep** (LOOP-03): Update best_metric and best_commit. Run:
     ```bash
     cd .ml && git add . && git commit -m "exp-{N}: {metric}={value} ({brief description})"
     ```
     Store the commit hash as best_commit.
   - **revert** (LOOP-04): Run:
     ```bash
     cd .ml && git checkout -- train.py
     ```
     IMPORTANT: Only revert train.py. Do NOT revert results.jsonl, experiments.jsonl, or checkpoint.json.
   - **retry** (LOOP-05): Edit train.py to reduce resource usage (smaller model, fewer estimators, lower batch size), then re-run. Increment retry counter.
   - **stop** (LOOP-06): Break the loop. Print reason (repeated OOM).

5. **Record results** (STATE-01, STATE-03, STATE-04):
   - Append to results.jsonl via python -c:
   ```bash
   python -c "
   from pathlib import Path
   from gsd_ml.results import ResultsTracker
   tracker = ResultsTracker(Path('.ml/results.jsonl'))
   tracker.add({
       'experiment': {N},
       'metric_name': '{metric}',
       'metric_value': {value},
       'kept': {True|False},
       'description': '{what was tried}'
   })
   "
   ```
   - Append journal entry via python -c:
   ```bash
   python -c "
   from pathlib import Path
   from gsd_ml.journal import JournalEntry, append_journal_entry, load_journal, render_journal_markdown
   entry = JournalEntry(
       experiment={N},
       metric_name='{metric}',
       metric_value={value},
       kept={True|False},
       description='{what was tried}',
       model_family='{model_family}',
       commit_hash='{commit_hash_or_empty}',
       timestamp='{ISO timestamp}'
   )
   append_journal_entry(entry, Path('.ml/experiments.jsonl'))
   entries = load_journal(Path('.ml/experiments.jsonl'))
   md = render_journal_markdown(entries)
   Path('.ml/experiments.md').write_text(md)
   "
   ```
   - Save checkpoint via python -c:
   ```bash
   python -c "
   from pathlib import Path
   from gsd_ml.state import SessionState
   from gsd_ml.checkpoint import save_checkpoint
   state = SessionState(
       run_id='{run_id}',
       domain='tabular',
       experiment_count={N},
       keep_count={keeps},
       revert_count={reverts},
       best_metric={best_metric_or_None},
       best_experiment={best_exp_or_None},
       best_commit='{best_commit_or_empty}',
       start_time='{start_time}'
   )
   save_checkpoint(state, Path('.ml'))
   "
   ```

6. Increment experiment_count. Continue loop.

**Phase 4: Finalize**

After loop ends (guardrail trip or all experiments done):

1. **Export best model** (FIN-01):
   ```bash
   python -c "
   import json
   from pathlib import Path
   from gsd_ml.export import export_artifact
   metadata = {
       'run_id': '{run_id}',
       'metric_name': '{metric}',
       'metric_value': {best_metric},
       'best_experiment': {best_experiment},
       'best_commit': '{best_commit}',
       'domain': 'tabular',
       'task': '{task}'
   }
   export_artifact(Path('.ml/best_model.joblib'), Path('.ml/artifacts'), metadata)
   "
   ```

2. **Generate retrospective** (FIN-02):
   ```bash
   python -c "
   import json
   from pathlib import Path
   from gsd_ml.results import ResultsTracker
   from gsd_ml.retrospective import generate_retrospective
   tracker = ResultsTracker(Path('.ml/results.jsonl'))
   results = tracker.load()
   config = {
       'run_id': '{run_id}',
       'domain': 'tabular',
       'task': '{task}',
       'metric': '{metric}',
       'direction': '{direction}',
       'dataset_path': '{dataset_path}',
       'target_column': '{target_column}',
       'experiments': results,
       'best_metric': {best_metric},
       'best_experiment': {best_experiment},
       'total_keeps': {keep_count},
       'total_reverts': {revert_count},
       'duration_minutes': {elapsed_minutes}
   }
   md = generate_retrospective(config)
   Path('.ml/RETROSPECTIVE.md').write_text(md)
   print('Retrospective written to .ml/RETROSPECTIVE.md')
   "
   ```

3. **Tag best commit** (FIN-03):
   ```bash
   git tag ml-best-{run_id} {best_commit}
   ```

4. **Final commit**:
   ```bash
   cd .ml && git add . && git commit -m "finalize: {run_id} complete, best {metric}={best_metric}"
   ```

5. **Print summary**: Display final stats (experiments run, keeps, reverts, best metric, time elapsed).
