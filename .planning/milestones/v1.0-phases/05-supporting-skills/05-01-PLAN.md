**Clean skill (commands/gsd-ml/ml-clean.md):**
- name: gsd:ml-clean
- description: Remove experiment state and optionally git branches
- argument-hint: "[--branches] [--force]"
- allowed-tools: [Read, Bash, Glob]
- execution_context points to ~/.claude/gsd-ml/workflows/ml-clean.md

**Clean workflow (gsd-ml/workflows/ml-clean.md):**

Step 1: Validate .ml/ exists. If missing, print "No experiment directory found." and stop.

Step 2: Preview. Show what will be removed:
- `du -sh .ml/` for total size
- `find .ml/ -type f | wc -l` for file count
- If --branches flag: also show `git branch --list 'ml/run-*'` and `git tag --list 'ml-best-*'`

Step 3: Offer artifact preservation. Check if `.ml/artifacts/` exists and contains files. If so, ask user: "Copy best model artifact to ./model/ before cleaning? (y/n)". If yes, `mkdir -p model && cp -r .ml/artifacts/* model/`.

Step 4: Confirm. Unless --force flag is present, ask user for confirmation: "Remove all experiment state? (y/n)". Wait for confirmation before proceeding.

Step 5: Delete.
- `rm -rf .ml/`
- If --branches: `git branch -D` each ml/run-* branch, `git tag -d` each ml-best-* tag
- Print summary of what was removed.

**Diagnose skill (commands/gsd-ml/ml-diagnose.md):**
- name: gsd:ml-diagnose
- description: Run diagnostics on the best model from last experiment
- argument-hint: ""
- allowed-tools: [Read, Write, Bash, Glob]
- execution_context points to ~/.claude/gsd-ml/workflows/ml-diagnose.md

**Diagnose workflow (gsd-ml/workflows/ml-diagnose.md):**

Step 1: Validate .ml/ exists and has checkpoint. Load checkpoint via load_checkpoint(). If no checkpoint or no best_commit, print "No experiment results found." and stop.

Step 2: Read config.json for task type, metric, domain.

Step 3: Check if .ml/predictions.csv already exists from last run. If it does, skip to Step 5 (use existing predictions). If not, proceed to Step 4.

Step 4: Generate predictions. Save current branch: `CURRENT=$(git rev-parse --abbrev-ref HEAD)`. Checkout best_commit's train.py: `git checkout {best_commit} -- .ml/train.py`. Run train.py to generate predictions.csv. Restore: `git checkout $CURRENT -- .ml/train.py`.

Step 5: Run diagnostics. Python one-liner:
- Read predictions.csv with pandas
- If task is classification/image_classification/text_classification: call diagnose_classification(y_true, y_pred)
- Else: call diagnose_regression(y_true, y_pred)
- Print results as JSON

Step 6: Format output. Print formatted markdown to terminal:
- "## Diagnostic Report"
- Worst predictions table (top 10 by error magnitude)
- Bias analysis (overall, per-class for classification)
- Feature-error correlations (if available)
- Confused class pairs (classification only)
- "## Suggested Actions" section: actionable hints based on findings (e.g., "High error on class X -> try class weighting or oversampling", "Large residuals on high values -> try log transform")

Per user decision: diagnose uses git checkout to get best_commit's train.py, then restores. Always save and restore current branch position. Requires .ml/ directory with prior experiment data.
