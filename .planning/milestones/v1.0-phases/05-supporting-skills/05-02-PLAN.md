**Resume workflow (gsd-ml/workflows/ml-resume.md):**

**Step 1: Validate .ml/ exists.** Check for `.ml/` directory and `.ml/checkpoint.json`. If missing, print "No experiment checkpoint found. Run /gsd:ml first." and stop.

**Step 2: Load checkpoint.** Python one-liner using load_checkpoint() to get SessionState. Print all fields as JSON. If --run <id> flag provided, verify the checkpoint's run_id matches (or look for the specified run in results.jsonl). If no match, error and stop.

**Step 3: Load config.** Read `.ml/config.json` to get domain, metric, direction, dataset_path, target_column, model_name, budget_experiments, budget_minutes, start_time.

**Step 4: Check budget.** Run check_guardrails() against the config. If budget is exhausted:
- Print current state: "{experiment_count} experiments run, budget was {budget_experiments} experiments / {budget_minutes} minutes"
- Ask user: "Budget exhausted. How many additional experiments would you like to add? (Enter a number, or 'cancel' to abort)"
- If user provides a number N: update config.json by adding N to budget_experiments. If time budget also exhausted, add proportional minutes. IMPORTANT: Do NOT reset start_time -- only increase budget_experiments and/or budget_minutes.
- If user cancels: stop.
- Write updated config.json back.

**Step 5: Checkout experiment branch.** Verify branch exists: `git branch --list "ml/run-{run_id}"`. If exists, `git checkout ml/run-{run_id}`. If not (e.g., branch was cleaned), create it from current HEAD: `git checkout -b ml/run-{run_id}`.

**Step 6: Print restoration summary.** Show what was restored:
```
Resuming experiment: {run_id}
  Domain: {domain}
  Metric: {metric} ({direction})
  Progress: {experiment_count} experiments ({total_keeps} keeps, {total_reverts} reverts)
  Best: {best_metric}
  Budget remaining: {budget_experiments - experiment_count} experiments
  Branch: ml/run-{run_id}
```

**Step 7: Re-enter experiment loop.** Instruct Claude to now follow `ml-run.md` starting at **Phase 3: Experiment Loop**, using the restored variables:
- All SessionState fields as loop variables
- Config values for guardrail checks
- The existing .ml/train.py as the starting point (already contains the best iteration's code on this branch)
- Do NOT re-run profiling or scaffolding (Phase 1 and 2 of ml-run.md)

Key pitfalls to document in the workflow:
- NEVER reset start_time when extending budget (per Phase 2 decision)
- ALWAYS checkout the correct branch before resuming (commits must land on experiment branch)
- Trust the checkpoint -- do not re-run the last experiment to verify
